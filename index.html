Welcome to Service Fabric I/O.

The service fabric enables Service Developers Rapidly develop Resilient Scalable applications Service Fabric is a framework that enables quick and easy development of modern distributed, resilient, scalable reactive applications such as on-line gaming, eTrading and Social. It greatly simplifies and streamlines distributed programming and provides a tool-set for managing micro-services architecture. 'Quick and easy' doesn't mean that a resulting application will suffer from a maintainability or a performance issue. Service Fabric has been designed carefully with the experiences earned from the implementation of a lot of Gaming Distributed Services. As a result, Service Fabric IO has succeeded to find a way to achieve ease of development, performance, stability, and flexibility without a compromise. The service fabric Features: -->

  • Simple, asynchronous programming model for writing distributed non-blocking services that can scale
  • Takes advantage of the JVM and scales over available cores 
  • Cluster aware and distributed Event Driven SOA
  • Technology abstraction with a plug and play model
  • Can be embedded in existing Java applications
  • Modular, Service is a stand alone deployable unit Service Fabric manages their topology on demand
  • Fault tolerance, node can appear or disappear at any given time.
  • Includes distributed Service Bus and Event bus
  • Zero Conf, Automatic discovery peer-to-peer service discovery using gossip.
  • Smart Routing and balancing strategies.

// Every Service has an interface, The Service interface is a simple java interface decorated 
// with annotation @Service annotation describes a logical name of the service. 
@Service(namespace = "my.service.namespace") // the service name
public interface IGameService {

    @ServiceMethod(datamodel = StartGameRequest.class, action = "startGame")        
    public void startGame(IContext context, Message requestMessage);

}

// The service implementation is a spring @Component implements a service interface. 
@Component
public class GameService implements IGameService {

   @Override
   public void startGame(IContext context, Message requestMessage) {
     // Log
     LOGGER.debug("Received start game request message: " + requestMessage);
   }
}

    How it works

    The service fabric node provision the services dynamically to the cluster so when a service request arrives to any service fabric node it is routed to the provisioning node in a transparent manner. that means that having service fabric nodes that provision no services makes these nodes transparent proxy and routers to the cluster. the nodes that do provision business services will serve the business request as they arrive to them but also may act as transparent proxy.
