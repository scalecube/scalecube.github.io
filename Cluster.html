Intro

Service Fabric Cluster provides a fault-tolerant decentralized peer-to-peer based cluster membership service with no single point of failure or single point of bottleneck. It does this using gossip protocols and an automatic failure detector.

Terms

Cluster Member

A logical member of a cluster. There could be multiple nodes on a physical machine. Defined by a member id.

Cluster

A set of nodes joined together through the membership service.

Seed Members

The seed members are configured contact points for new members joining the cluster. When a new member is started it sends to all seed nodes (one by one) sync command in order to synchronize membership tables until one of them answers, if all failed to answer it is decided that this member is the first member in the cluster. Seed members will be used further in order to restore cluster from network partitioning They will continued to be and sync with cluster nodes even if they are marked as failed. So it is possible to configure any node as seed member in order to join the cluster, but it is recommended to configure nodes with most probability to work all the time so in such case they can restore from network partitioning via this seed members (e.g. possible choice for seed members are API Gateways) The seed members configuration value does not have any influence on the running cluster itself, it is only relevant for new members joining the cluster as it helps them to find contact points to send the join command to; a new member can send this command to any current member of the cluster, not only to the seed members. The seed nodes can be started in any order and it is not necessary to have all/or any seed nodes running, but when initially starting a cluster, the cluster will be initialized only when one or more seed-nodes will become initialized else no other node can join the cluster and nodes can only join themselves. It is quickest to start all configured seed nodes at the same time (order doesn't matter), otherwise it can take up to time until the nodes can join. Once more than two seed nodes have been started it is no problem to shut down the first seed node. If the first seed node is restarted, it will first try to join the other seed nodes in the existing cluster. You can join to any node in the cluster. It does not have to be configured as a seed node. Note that you can only join to an existing cluster member, which means that for bootstrapping some node must join itself,and then the following nodes could join them to make up a cluster.

Failure Detector

The failure detector is responsible for trying to detect if a node is unreachable from the rest of the cluster

Preparing Your Project for Clustering

The Service Fabric I/O cluster is a separate jar file. Make sure that you have the following dependency in your project:

    <dependency>
      <groupId>io.servicefabric</groupId>
      <artifactId>servicefabric-cluster<artifactId>
      <version>${project.version}<version>
    </dependency>

Simple Cluster examples

setting up cluster nodes A and B while A is seed node and B use A as seed: 
    // create cluster member A on port 3000 and join as seed member
    ICluster clusterA = Cluster.newInstance(3000).join();

    // create cluster member B on port 3001 and join as seed member A
    ICluster clusterB =  Cluster.newInstance(3001,"localhost:3000").join();

Send gossip from member-A that eventually will reach all members

Subscribe to gossip: 
    // subscribe to all gossip messages:
    clusterA.gossip().listen().subscribe(new Action1() {
        @Override
        public void call(Message gossip) {
            // print out the gossip message
            System.out.println("Gossip message:"  + gossip);
        }
    });
spread gossip from member-B: 
    // spread gossip
    clusterB.gossip().spread(new Message(new Greetings("Greetings from ClusterMember B")));

Send message from member-B to member-A

Subscribe to hello world message on member-A using filter: 
    // Filter and subscribe to greetings messages:
    clusterA.listen().filter(new Func1() {
        @Override
        public Boolean call(TransportMessage t1) {
            return t1.message().data() != null && Greetings.class.equals(t1.message().data().getClass());
        }
    }).subscribe(new Action1() {
        @Override
        public void call(TransportMessage t1) {
            System.out.println(t1.message().data());
        }
    });
Send hello world message from member B to member-A: 
    // Send greeting message to other cluster members
    List members = clusterB.membership().members();
    ClusterMember localMember = clusterB.membership().localMember();
    for (ClusterMember member : members) {
        if (!clusterB.membership().isLocalMember(member)) {
            clusterB.to(member).send(new Message(new Greetings("Greetings from ClusterMember B")));
        }
    }
